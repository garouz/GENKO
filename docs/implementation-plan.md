# フェーズ1実装計画

**目標**: ルールベース + 用語辞書照合によるコア校正機能の実装

---

## 📋 実装スコープ

### ✅ フェーズ1で実装する機能

1. **原稿管理**
   - Word(.docx)ファイルアップロード
   - 原稿プレビュー表示
   - メタデータ管理(タイトル、著者など)

2. **用語辞書管理**
   - CSV/JSONファイルのインポート
   - 辞書データベース構築(IndexedDB)
   - 辞書の表示・管理UI

3. **AI自動チェック(ルールベース)**
   - 誤字脱字検出(簡易版)
   - 表記ゆれ検出(用語辞書照合)
   - チェック結果の構造化

4. **チェック結果の提示**
   - 原稿テキストのハイライト表示
   - 指摘事項リスト表示
   - フィルタリング・ソート

5. **編集者レビューUI**
   - 反映/非反映の判定
   - コメント追加機能
   - 修正プレビュー

6. **ファイル出力**
   - Word(.docx)形式の出力
   - PDF形式の出力
   - Markdown形式の出力

7. **PWA化**
   - Service Workerの実装
   - オフライン対応(簡易版)
   - インストール機能

### ❌ フェーズ1では実装しない機能

- ユーザー認証・認可(簡易版ユーザー情報で進行)
- チーム・権限管理
- LLM API統合
- 校正履歴・バージョン管理
- バックアップ・災害対応

## 🔧 技術的検討事項

1. **ルールベース誤字脱字検出**
   - 日本語テキスト処理ライブラリの選定(MeCab, Janome等)
   - または正規表現ベースの簡易実装

2. **用語辞書照合の精度**
   - 完全一致 vs 部分一致の使い分け
   - 複合語への対応

3. **パフォーマンス最適化**
   - 大型ファイル(10MB以上)への対応
   - キャッシング戦略

4. **UI/UXの最適化**
   - ハイライト表示の視認性
   - リスト表示のスクロール・レスポンシブ対応

## 📊 成功基準

- ✅ 5MB未満の.docxファイル読み込み・チェック・出力が動作する

- ✅ 誤検出率が20%以下に抑えられている
- ✅ 編集者が直感的に修正・確認できるUI
