# 社内校正システム 要件定義書

- 作成日: 2026-07-09
- バージョン: v0.1 → v0.2（実装開始）
- ステータス: 実装中

---

## 1. 概要

医師・看護師等の医療関係者から受領する、看護師・看護学生向け雑誌・書籍の原稿に対して、AIによる自動チェック（ルールベース + LLM）と編集者による人的レビューを組み合わせた校正作業を支援する社内向けPWA(Progressive Web App)。

## 2. 背景・目的

- 現在、原稿校正は編集者の手作業に依存しており、誤字脱字や医療専門用語の表記ゆれ、看護師国家試験基準に沿った用語統一などのチェックに時間と労力がかかっている。
- 社内で保有する用語辞書を活用したAI自動チェック（ルールベース優先、後にLLM統合）を導入することで、編集者の一次チェック負荷を軽減し、レビュー品質と作業効率を向上させる。
- まずは校正コア機能に絞った小さなプロトタイプとしてリリースし、実運用を通じて段階的に機能を拡張する。

## 3. 対象ユーザー

| 項目 | 内容 |
|---|---|
| 利用者 | 社内編集者(全社、数百名規模) |
| 権限・チーム分け | フェーズ1では簡易版（全ユーザー全案件アクセス可能）、フェーズ2で雑誌・書籍単位のチーム分け |
| 外部関係者(医師・看護師等の執筆者) | システムは利用しない。原稿受領・フィードバックはシステム外(メール等)で実施 |

## 4. 校正フロー(全体像)

```
① 原稿受領(.docx)
      ↓
② AI自動チェック(用語辞書ベース + ルールベース)
      ↓
③ 編集者レビュー(AI指摘の確認・修正・コメント)
      ↓
④ 校正済みファイル出力(Word/PDF/Markdown)
      ↓
⑤ 編集者が出力ファイルを用いて執筆者へフィードバック(システム外)
```

## 5. 機能要件

### 5.1 原稿インポート
- Word形式(.docx)の原稿をアップロードできる
- 原稿のメタデータ(タイトル、著者、作成日)を管理
- アップロード履歴・バージョン管理(フェーズ1では簡易版)

### 5.2 用語辞書管理
- 社内で保有する用語辞書(Excel/CSV/JSON形式)をインポートできる
- インポートした用語辞書はAI自動チェックの照合元として使用
- 辞書の追加・更新ができる(運用しながら随時メンテナンス想定)
- 辞書フォーマット: {"正規用語", "表記ゆれ候補", "分野", "説明"}

### 5.3 AI自動チェック

**フェーズ1: ルールベース + 用語辞書照合**
- 誤字脱字・表記ゆれの検出(ルールベース)
- 看護専門用語・国家試験基準に沿った用語統一チェック(用語辞書照合)
- 基本的な医療情報の事実関係チェック(ルール定義)

**フェーズ2以降: LLM統合**
- ChatGPT/Claude等のLLM APIを活用した高度なチェック
- 自動チェック結果の精度向上

### 5.4 チェック結果の提示
- 原稿上の該当箇所をハイライト表示し、修正提案コメントを添える
- 指摘事項を位置・種別(誤字/表記ゆれ/用語/その他)・内容・重要度ごとに一覧リストとしても表示
- ハイライトと一覧の両方を提供(双方向リンク可能)

### 5.5 編集者レビュー
- AIの指摘に対し、編集者が「反映する/しない」を判断し、修正を加えられる
- 編集者自身によるコメント・修正の追記ができる
- 修正履歴の記録

### 5.6 アウトプット
- 校正済みの原稿を、Word・PDF・Markdownの3形式で出力(ダウンロード)できる
- 出力時に修正内容を確認・調整可能

### 5.7 権限・チーム管理
- **フェーズ1**: ユーザー認証(簡易版)
- **フェーズ2**: 雑誌・書籍単位で担当編集チームを分け、担当外の原稿は閲覧・編集できないようにする

## 6. 非機能要件

| 項目 | 内容 |
|---|---|
| 利用環境 | PCブラウザでの利用を想定 |
| ネットワーク | 常に社内ネットワーク上での利用を前提とし、オフライン対応は不要 |
| セキュリティ | 医療系の原稿・情報を扱うため、社内ネットワーク限定でのアクセス制御を行う |
| 対応ブラウザ | 主要モダンブラウザ(Chrome, Edge, Firefox等)を想定 |
| パフォーマンス | 原稿AI自動チェック: 5MB未満のファイルで30秒以内に完了 |
| ストレージ | 原稿・辞書データをクライアント側 IndexedDB + サーバー側DBで管理 |

## 7. 技術スタック

### フロントエンド
- React 18 + TypeScript
- Vite(ビルドツール)
- TailwindCSS(UI)
- Service Worker(PWA化)

### ファイル処理
- mammoth.js(.docx読み込み)
- jsPDF/pptx-gen-js(出力)
- papaparse(CSV解析)
- markdown-it(Markdown出力)

### バックエンド
- Node.js + Express
- TypeScript
- cors, body-parser等のミドルウェア

### AI処理
- **フェーズ1**: ルールベース + 用語辞書照合(自実装)
- **フェーズ2**: LLM API(OpenAI/Anthropic等、要件別に選定)

### ストレージ
- IndexedDB(クライアント側)
- SQLite/PostgreSQL(サーバー側、後日検討)
